# 科技 风演示文稿生成器 - 详细需求规格

## 1. 核心目标
- 开发一个 **单文件 HTML PPT 生成器**，采用科技/风格美学，面向移动端和现代演示场景
- 输出 9:16 纵向比例的演示文稿
- 保持所有资源（HTML、CSS、JS）打包在单个文件中，无需构建步骤
- 使用 TailwindCSS 通过 CDN 引入，实现快速样式迭代

## 2. 文件结构

```
ppt-generator.html          # 主应用文件（包含 HTML、CSS、JS）
ppt-generator-requirements.md  # 本规格文件
```

## 3. 幻灯片类型

| 类型 | 说明 | 示例输入 |
|------|------|----------|
| **cover** | 封面页，包含标题、副标题、日期 | `封面标题：你的演示标题`<br>`副标题：演讲者或副标题信息`<br>`日期：2026 年 3 月` |
| **title** | 章节标题页 | `第一章：章节标题`<br>`章节简要说明`<br>`（可选）描述性文字` |
| **quote** | 引言/引用页 | `引用：引用内容`<br>`- 作者` |
| **steps** | 步骤列表页 | `步骤：步骤标题`<br>`步骤项目：项目内容` |
| **ending** | 结束页 | `结束页信息：感谢观看`<br>`联系方式：contact@example.com` |

## 4. 输入内容格式

> **注意**：以下示例使用中文标识符，实际使用时可自定义语言/标签。

```
封面标题：未来之路
副标题：科技趋势分析
日期：2026 年 3 月

第一章：人工智能的崛起
AI正在改变各行各业

引用：人工智能是未来的通用技术
- 作者姓名

第二章：量子计算的突破
量子计算将解决传统计算无法处理的问题

步骤：关键技术
步骤项目：人工智能
步骤项目：云计算
步骤项目：大数据

第三章：可持续能源的革命
可再生能源正成为新的能源支柱

结束页信息：感谢观看
联系方式：contact@example.com
二维码：/path/to/qr.png
```

### 关键语法规则
- **章节标题** 必须以 `第一章：`、`第二章：`、`第三章：` 等前缀开头
- **引用** 必须以 `引用：` 开头，后跟内容行，随后可选的作者行
- **步骤** 必须以 `步骤：` 开头，后跟步骤标题；每个实际项目必须以 `步骤项目：` 开头
- **结束页** 必须以 `结束页信息：` 开头，后跟主信息；随后可包含 `联系方式：`、`二维码：` 等附加信息

## 5. 功能特性

### 5.1 幻灯片渲染
- 每种类型的幻灯片都有专属的样式（背景、标题、布局）
- 封面页：渐变背景 + 大标题 + 副标题 + 日期
- 标题页：章节编号 + 章节标题 + 描述
- 引用页：左侧引用块 + 著作归属
- 步骤页：编号步骤 + 内容区
- 结束页：结束语 + 联系方式 + QR 占位

### 5.2 导航
- 点式导航（底部圆点）对应每页
- 键盘箭头键/左右滑动切换
- 移动端手势滑动（实现 inertia）

### 5.3 动画
- 每页入场动画（CSS keyframes）
- 步骤列表项目可渐显

### 5.4 主题切换
- 深色/浅色主题切换（默认深色）
- 主题切换按钮持久化至 `localStorage`

### 5.5 导出
- “下载 HTML”按钮，打包当前幻灯片为单文件 HTML
- 可选：浏览器打印为 PDF（利用 `window.print()`）

## 6. 设计与实现约束

- **单文件**：所有 CSS、JS 必须内联在 `<style>` 与 `<script>` 中
- **TailwindCSS**：仅通过 CDN 引入，不使用本地构建工具
- **响应式**：兼容手机、平板、桌面浏览器
- **skill选额**：充分利用frontend-design等Claude Code skill
- **可维护性**：使用 CSS 变量管理颜色主题，保持一致性
- **可访问性**：使用语义化 HTML，确保键盘可操作

## 7. 关键实现点（开发者参考）

| 功能 | 关键函数/元素 | 备注 |
|------|---------------|------|
| 解析输入 | `parseContent(content)` | 使用正则匹配章节、引用、步骤等结构 |
| 渲染幻灯片 | `getSlideHTML(slide)` | 根据 `slide.type` 返回不同 HTML 模板 |
| 更新视图 | `renderSlides()` | 重新生成所有幻灯片 DOM |
| 导航控制 | `updateSlide(index)` | 根据索引显示对应幻灯片，更新导航圆点 |
| 键盘/触摸 | `handleKeyDown()` | 处理 `ArrowUp/Down`, `Home`, `End` |
| 主题切换 | `theme-toggle` 事件 | 保存至 `localStorage` 并应用 CSS 变量 |
| 导出 HTML | `exportToFile()` | 创建 Blob 并触发下载 |

## 8. 样式规范

- 采用 **9:16** 视口比例，确保移动端全屏展示
- 使用 **TailwindCSS** 实用类快速构建布局
- 自定义 CSS 变量定义：
  ```css
  --bg-primary: #0f172a;
  --bg-secondary: #1e293b;
  --accent: #60a5fa;
  --light: #f1f5f9;
  ```
- 动画关键帧：
  ```css
  @keyframes ambient-1 { ... }
  @keyframes ambient-2 { ... }
  @keyframes ambient-3 { ... }
  ```

## 9. 测试与验证

1. **手动测试**：在浏览器打开 `ppt-generator.html`，输入示例内容，检查每类幻灯片渲染效果。
2. **自动化断言**（可选）：使用浏览器开发者工具执行简单的断言脚本，验证关键元素（如 `.slide-type-cover` 是否存在）是否正确渲染。
3. **跨设备检查**：在不同尺寸的设备或模拟器上打开，确保 9:16 比例和触控交互正常。
